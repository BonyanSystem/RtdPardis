EventLink Temporary Record Storage Library

EventLink Temporary Record Storage Library is used for storing and retrieving event records and the related data into an in-memory relational database. EventLink Temporary Record Storage Library can be used, for example, in building business logics for customised aggregation and correlation.

Functionality of EventLink Temporary Record Storage Library

EventLink Temporary Record Storage Library is used to enhance business logic and to develop products requiring short-term data storage. EventLink Temporary Record Storage Library has a configuration tool that is used to create the data store and tables. The configuration tool can also be used for exporting, importing and migrating the data.

The library provides two interfaces:

·         SQL interface for direct access to the database

·         Temporary Record Storage interface for storing, retrieving, updating and deleting event records in the database

EventLink Temporary Record Storage Library connects to a data store given as a configuration parameter. Multiple EventLink Temporary Record Storage Libraries can access the same data store. The library or configuration tool creates tables and indexes when the data store is accessed for the first time. If the data store and tables already exist, the library checks that the tables match the current configuration. If there is a mismatch, the library does not initialise and reports an error.

If the configuration is changed after a node or configuration tool has been run, the data store has to be migrated to the new configuration. If the migration is simple, it can be done with export/import tools, otherwise by running custom sql scripts.

Each type of record to be stored has its own table, as described in the configuration file. A table has key fields, which are indexed, and data fields, which are not. The application can query the values of the fields by the index number or field name.

Configuration of EventLink Temporary Record Storage Library

ODBC configuration file

The data stores used for storing data in EventLink Temporary Record Storage can be specified in the ODBC configuration file .odbc.ini. The file is assumed to reside by default in user's home directory. To override the default file name and location, the ODBCINI environment variable can be used.

Configuration parameters

The following table lists the EventLink Temporary Record Storage Library configuration parameters. In the M/O/C column, M stands for a mandatory option, O for an optional and C for a conditional one.

Table 1. EventLink Temporary Record Storage Library configuration parameters

 

Parameter

Description

Values

M/O

TRS.DataStorage

The name or path and name of the data storage.

This parameter is mandatory if TRS.ConnectString is not defined.

If the data store is specified in the ODBC configuration file, the TRS.DataStorage parameter can be used.

Example value:

/comptel/elink/datastores/aggrDS

C

TRS.DataStorageSize

The size to be allocated for the data storage in megabytes.

The data stores cannot shrink, but they grow if the value of this parameter is larger than the current data store size and EventLink Temporary Record Storage Library is the first client to connect to the data store.

If the parameter is omitted, the TimesTen default value is used.

Example value:

5

O

TRS.ConnectString

The connect string to be used when connecting to the data storage.

This parameter overrides the default settings.

The TRS.ConnectString parameter can be used to define the data store which is not in the ODBC configuration file. It can also be used to override the default values. If both TRS.DataStorage and TRS.ConnectString are defined, TRS.ConnectString is used and the other values are ignored.

Example value:

Driver=/comptel/elink/TimesTen/SunOS/lib/libtten.so;DataStore=
/comptel/elink/datastorages/aggregationDS;DSN=aggregationDS

O

TRS.UseScalability

The handling of the node scalability.

If the parameter has value 1, EventLink Temporary Record Storage Library automatically handles the database access when multiple nodes are run in parallel to prevent deadlocks. If the value is 0 (zero) or the parameter is omitted, parallel access and deadlock handling should be handled by the application, if necessary.

Possible values:

0, 1

O

 

 

 

Configuration examples

Example 1:

The following gives an example of a data store that is specified in the ODBC configuration file. Data storage size is given to override the default value. Scalability is handled automatically.

DataStorage            aggregationDS
DataStorageSize       200
UseScalability           1

Example 2:

The following gives an example of the data store setting specification. No ODBC configuration file is needed. Scalability is not handled by the library.

ConnectString           Driver=/comptel/elink/TimesTen/SunOS/lib/libtten.so;DataStore=/comptel/elink/datastorages/aggregationDS;DSN=aggregationDS
